How it works

Instrument once. Inspect everything.

Three concepts — Tracer, Span, Eval. That's the whole API.

Your agent code — instrument with context managers:

from agentlens import Tracer, evals as E

tracer = Tracer()

with tracer.trace("shopping-agent", user="alice"):
  with tracer.agent("router"):

    # LLM call — tokens + cost auto-tracked
    with tracer.llm("classify", model="gpt-4o-mini") as s:
      s.record_tokens(prompt=120, completion=12,
                      model="gpt-4o-mini")
      s.set_output(intent="order_status")

    # Tool call — inputs + outputs captured
    with tracer.tool("lookup_order", id="A1029") as s:
      s.set_output(status="shipped", eta="2 days")

    # Event marker — zero-duration checkpoint
    tracer.event("guardrail_passed")

tracer.save("runs/")  # → runs/<trace_id>.json

Eval Gates

Behavior as code. Fail the build, not the customer.

9 built-in checks. Drop assert report.passed in pytest. CI breaks before customers notice drift.

from agentlens import evals as E

suite = E.Suite("guardrails", [
  E.succeeded(),
  E.called_tool("lookup_order"),
  E.never_called_tool("issue_refund"),
  E.tool_before("lookup_order", "send_reply"),
  E.max_cost(0.01),
  E.max_tokens(500),
  E.max_duration_ms(5000),
  E.had_event("human_approved"),
  E.custom("reply has order id",
    lambda t: "A1029" in str(t.spans)),
  E.llm_judge(
    "Reply is accurate and friendly",
    threshold=0.7,
  ),
])

report = suite.run(trace)
assert report.passed  # ← in pytest

Honest gaps

What's not available yet

We'd rather tell you upfront than have you find out in production.

✗ No async / concurrent agents

The Tracer uses a single stack. Two parallel asyncio tasks will collide.

Workaround: create one Tracer per concurrent run.

✗ No real-time streaming traces

Spans flush at the end of the run. Can't watch a trace build live during token streaming.

Workaround: inspect traces post-run with agentlens ls.

✗ No auto-patching of OpenAI SDK

Doesn't monkey-patch openai.ChatCompletion. Manual wrapping required unless using an adapter.

Workaround: use the OpenAI Agents SDK adapter, or wrap manually.

✗ No PII detection / redaction

Inputs and outputs are stored as-is. No built-in scrubbing of names, emails, or card numbers.

Workaround: call s.set_input(text=redact(text)) before recording.

✗ No alerts or webhooks

No "email me when cost exceeds $X" or Slack notifications built in.

Workaround: check report.passed in your own alerting code.

✗ No browser / JS SDK

Python only. No frontend tracing, no React integration, no browser spans.

Workaround: send custom events from your backend to AgentLens.
